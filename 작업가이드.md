#### 언제 수정해야 하나요?

다음과 같은 게 바뀌었을 때 **꼭** `.readme_ecbt.md` 파일을 수정하세요:

1. ✅ **비밀번호나 키 같은 보안 정보가 바뀜**

   - Supabase API 키 바뀜

4. ✅ **운영 방법이 바뀜**

## 📝 수정하는 방법

### 1. 파일 위치 확인
```
프로젝트 폴더/
└── .readme_ecbt.md  (숨김 파일)
```

### 2. 파일 열기
```bash
# VS Code로 열기
code .readme_ecbt.md

# 또는 메모장으로 열기
notepad .readme_ecbt.md
```

### 3. 관련 부분 찾아서 수정하기
- 비밀번호/키 정보 → "🔐 보안 정보" 부분
- 서비스 주소 정보 → "📍 서비스 정보" 부분
- 데이터베이스 정보 → "📊 데이터베이스 구조" 부분
- 운영 방법 → "🔧 운영 가이드" 부분
- 문제 해결 방법 → "🐛 문제 해결 가이드" 부분

### 4. 수정 날짜 바꾸기
파일 맨 아래의 "마지막 업데이트" 날짜를 바꾸세요:
```markdown
**마지막 업데이트**: 2025-01-15
```

## 📤 Git에 올리는 방법

### 1. 변경된 파일 확인
```bash
git status
```

### 2. 변경된 파일 추가하기
```bash
# 모든 변경된 파일 추가
git add .

# 또는 특정 파일만 추가
git add src/pages/Login.tsx
git add src/services/storage.ts
```

### 3. 커밋하기
```bash
# 간단한 메시지
git commit -m "기능 추가: 로그인 시스템 개선"

# 또는 자세한 메시지
git commit -m "로그인 시스템 개선: 게스트 모드 제거, 이름/전화번호/이메일 로그인으로 변경"
```

### 4. GitHub에 푸시하기
```bash
git push origin main
```

### 5. Vercel 자동 배포 확인
- GitHub에 푸시하면 Vercel이 자동으로 배포를 시작합니다
- Vercel 대시보드에서 배포 상태를 확인하세요
- 배포 완료 후 https://www.ecbt.kr 에서 테스트하세요

### ⚠️ 주의사항

1. **`.readme_ecbt.md` 파일은 커밋하지 마세요**
   - `.gitignore`에 포함되어 있어 자동으로 제외됩니다
   - 실수로 추가하려고 하면 경고가 표시됩니다

2. **민감한 정보가 포함된 파일 확인**
   - API 키, 비밀번호 등이 코드에 직접 들어가 있지 않은지 확인
   - 환경 변수로 관리되는지 확인

3. **커밋 전 테스트**
   - 로컬에서 `npm run dev`로 테스트
   - 빌드 테스트: `npm run build`
